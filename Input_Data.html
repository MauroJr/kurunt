Input Data

Your data can be inputed many ways through Kurunts TCP, UDP and HTTP inputs. These inputs allows you to use your own clients sending your data through these protocols. Below are some examples of different ways of sending data to Kurunt. Note that the data you send should be in the format of the coresponding 'worker' for your stream.

Curl (http input only)

You can use GET, PUT or POST to send data through the http input.

curl -X POST|PUT|GET -d 'YOUR_MESSAGE' INPUT_ADDRESS

For example, sending JSON data.

curl -X POST -H "Content-Type: application/json" -d '{"hello":"world"}' http://127.0.0.1:5555/fde5554c4a7b00cf

Netcat (tcp input)

echo 'hello world' | netcat -w 1 127.0.0.1 6001

Netcat (udp input)

echo 'hello world' | netcat -u -w 1 127.0.0.1 6001

Web browser/client (http input only)

GET, can optionally use a .gif file extention, will return a 1 pixel transparent gif.

INPUT_ADDRESS/?KEY=VALUE

For example:

http://127.0.0.1:5555/fde5554c4a7b00cf/pic.gif?hello=world

Kurunt Client Module (all inputs)

You can use the cli as shown bellow to send data or use the node Kurunt Client Module (npm install kurunt-client) to incorporate sending data into your own projects.

node kurunt/client/send.js -apikey=APIKEY -data=YOUR_MESSAGE

-data should be in the format of the 'worker', for example using JSON:

node kurunt/client/send.js -apikey=6001 -data='{"hello":"world"}'

Perl benchmarking (tcp or udp inputs)

If you want to benchmark Kurunts performance at processing messages you can use the perl benchmarking client. Use -help command to see options.

perl kurunt/client/send.pl -help

For example sending 100 json messages over 10 seconds through a tcp input:

perl kurunt/client/send.pl -T=tcp -apikey=6001 -m=100 -c=10 -d='{"hello":"world"}'
